## Mapping to runtime architecture

The project layout tracks runtime subsystem boundaries:

| Directory | Runtime mapping |
| --- | --- |
| `application.ex` | Supervision tree root |
| `catalog/` | Central registry and persistence boundary |
| `consumer/` | Source consumption concerns |
| `producer/` | Sink delivery concerns |
| `api/` and `mcp/` | External interfaces |
| `schemas/` | Ecto persistence types and validation |
| `telemetry.ex` | Observability hooks |

## Key dependencies

| Dependency | Purpose |
| --- | --- |
| `plug` | HTTP abstraction and request pipeline |
| `bandit` | HTTP server |
| `ecto_sql` + `postgrex` | PostgreSQL access and migrations |
| `phoenix_pubsub` | Internal event bus |
| `req` | HTTP client for adapter fetches |
| `floki` | HTML parsing for content extraction |
| `jason` | JSON encoding and decoding |
| `cloak_ecto` | Credential encryption at rest |
