## Phase 1: single release

Elixir compiles to a **release**: a self-contained directory containing
the BEAM VM, compiled bytecode, and all runtime dependencies.

This means:

- No separate Elixir installation required on the target machine
- A single container deployment is straightforward
- PostgreSQL is the primary external dependency
- The HTTP layer runs as a Plug application served by Bandit

Runtime configuration (database URLs, secrets) is loaded from
`config/runtime.exs` at boot, not compile time. The same release
artifact works across environments by changing environment variables.

## Phase 2: distributed Erlang

If Claptrap ever needs to run across multiple machines, the BEAM has
built-in node clustering:

- Nodes can be connected directly
- Phoenix.PubSub can distribute messages across Erlang nodes
- Ecto connections remain node-local, with each node maintaining its
  own database pool

## Scaling posture

The design is intentionally conservative. It does not optimize for
horizontal scale first. It optimizes for:

- Operational simplicity
- Correct failure isolation
- A personal-tool deployment model

A single-node release with PostgreSQL is enough until real evidence
says otherwise.
